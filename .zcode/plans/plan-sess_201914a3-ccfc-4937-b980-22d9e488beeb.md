# OfficeSceneV2 — Visual Director Pass (parallel scene)

**Meta:** reemplazar visualmente la escena central por una oficina isométrica compacta, densa y premium. Misma lógica de agentes/movimiento/Live Mode — solo cambia el render del mundo.

## Plan escrito (lo que se va a reemplazar y cómo)

**Se deja de usar visualmente:** `OfficeFloor.tsx` (piso/zonas/clip-paths viejos), `FurnitureLayer.tsx` (shapes dispersos), `MonitorGlow.tsx` (integrado en V2), `officeZones.ts` rects viejos, `officeLayout.ts` walls/spots viejos, `furniture.ts` piezas viejas. **No se borran** — quedan en el repo por si acaso, pero OfficeWorld ya no los renderiza.

**Se mantiene intacto (renderer-agnostic, no se toca):** `AgentMotionStore`, `scheduleActivities`, `Agent` type, `INITIAL_AGENTS`, `tick/simulateHour`, `CodexPetSprite`, `AgentSprite` (solo se le pasa un `SceneApi` nuevo para proyección), `isometric.ts` math, App.tsx wiring (Live Mode interval sin cambios).

## 1. Layout curado — `src/data/officeSceneV2Layout.ts`

Una grilla **compacta de 22×18** (vs 30×24 actual) para que la oficina ocupe más pantalla y se vea más densa. Coordenadas manuales e intencionales, no procedurales.

**8 zonas con rectángulos curados:**
- Meeting/Strategy Room: (0,0,6,6) — superior izquierda, con glass partitions
- Reception/Lobby: (6,0,5,5) — superior centro, entrada visible
- Open Workspace: (6,5,8,6) — centro, corazón de la oficina, 8+ estaciones
- Engineering Pods: (14,0,8,7) — derecha, 3 pods con dual monitors
- Research/QA Area: (0,6,6,7) — izquierda, shelves + test benches + boards
- Finance/Legal Corner: (0,13,6,5) — inferior izquierda, formal
- Break Area: (6,11,4,7) — inferior centro, sofá/café
- Command Center: (10,11,12,7) — inferior derecha, zona WOW (wall de 6 pantallas)

Cada zona exportada con `id, name, rect, color (tenue), accent (para glow)`. Los colores son sutiles — las zonas se distinguen por **mobiliario**, no por pintura.

## 2. Anclas pegadas a muebles — `src/data/officeSceneV2Anchors.ts`

**28+ activity anchors**, cada uno con `nearbyFurnitureId` explícito (validación: ningún anchor sin mueble cercano). Los agentes se mueven hacia estos anchors. `scheduleActivities` se adapta para usar estos anchors en vez de los viejos.

```ts
type OfficeSceneV2Anchor = {
  id: string; zoneId: string; x: number; y: number;
  activity: "desk-work"|"engineering"|"meeting"|"command-center"|"research"|"qa"|"finance-legal"|"break"|"reception";
  nearbyFurnitureId: string; preferredRoles: string[];
};
```

## 3. Mobiliario semántico grande — `src/components/iso/`

Componentes dedicados **más grandes y reconocibles** (no rectángulos pequeños):
- `IsoDesk` (superficie + patas + sombra + monitor integrado opcional)
- `IsoChair` (silla visible, no un punto)
- `IsoDualMonitorDesk` (estación de ingeniería con 2 monitores azules)
- `IsoMeetingTable` (mesa grande 3×2 con vetas de madera)
- `IsoReceptionDesk` (mostrador grande con letrero "AGENT OFFICE")
- `IsoCommandWall` (muro de 6 pantallas grandes con glow azul — el punto WOW)
- `IsoSofa`, `IsoCoffeeMachine`, `IsoBookshelf`, `IsoWhiteboard`, `IsoServerRack`, `IsoFilingCabinet`, `IsoPlant`, `IsoGlassPartition`, `IsoRug`, `IsoLamp`

Cada uno es CSS/HTML simple pero con suficiente detalle (sombras, bordes, glow, vetas) para leerse como objeto real, no placeholder.

## 4. Capas de la escena — `OfficeSceneV2.tsx`

Reemplaza a OfficeWorld como escena principal. Mantiene el **mismo patrón de movimiento** (posee `AgentMotionStore` ref, `sync()` cada render, rAF `step()` loop, `frame` counter para bust memo). Capas:

1. **`OfficeSceneV2Floor.tsx`** — piso base cálido (no negro plano), con textura sutil de baldosas, rugs por zona, y bordes suaves. Sin clip-paths de zona saturados.
2. **`OfficeSceneV2Zones.tsx`** — tinte de zona muy sutil (~15% opacidad) + señalética pequeña opcional. Las zonas se leen por muebles.
3. **`OfficeSceneV2Furniture.tsx`** — renderiza los componentes `Iso*` desde el layout V2, depth-sorted por `x+y`.
4. **`OfficeSceneV2Lighting.tsx`** — glow focal: command center azul, monitores encendidos, lámparas cálidas. Integra el MonitorGlow.
5. **AgentSprite** (reutilizado) — proyectado en la misma grilla V2.

## 5. Conexión de movimiento a anchors V2

- `scheduleActivities` se actualiza para leer de `officeSceneV2Anchors` (mismas firmas `pickFreeAnchor`/`anchorOccupiedKey`).
- Los waypoints siguen funcionando (corridor midpoint para movimientos largos).
- Los agentes arrancan en sus home spots V2 (agentes.ts seeds actualizados a posiciones V2).
- Validación: agentes siguen caminando suavemente, llegan a muebles reales, no teleport.
